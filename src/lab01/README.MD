## Вопрос 1. Что является сущностью?

Игровой персонаж (Player) в многопользовательской игре.

Это активная единица игрового мира
Имеет прогрессию (уровни, опыт).
Взаимодействует с игровым миром (получает урон, набирает опыт).
Принадлежит конкретному игроку (уникальный никнейм).

## Вопрос 2. Какие у него атрибуты?

Характеристики (состояние):

nickname - уникальное имя персонажа
player_class - класс персонажа (warrior, mage, archer, healer)
level - текущий уровень развития
health - количество здоровья
experience - накопленный опыт
Действия (методы):

take_damage() - получить урон.
gain_experience() - получить опыт.
is_alive() - проверка состояния.

## Вопрос 3. Какие инварианты?

Никнейм:
- Не пустой (минимум 3 символа)
- Не слишком длинный (максимум 20 символов)
- Уникальный в системе (подразумевается, но не реализовано)
*Класс персонажа:*
Должен быть из предопределенного списка available_classes
Уровень:
Не меньше 1
Не больше max_level (100)
Здоровье:
Не отрицательное
При health = 0 персонаж считается мертвым
Опыт:
Не отрицательный
При достижении порога повышается уровень
Связь уровень-здоровье:
При повышении уровня здоровье увеличивается на 20

## Вопрос 4. Что значит "равенство"?

Два игрока равны, если совпадают:

Никнейм (уникальный идентификатор игрока)
Класс персонажа (игрок может иметь несколько персонажей разных классов)
При этом уровень, здоровье и опыт могут отличаться - это изменяемые характеристики одного персонажа.

## Вопрос 5. Есть ли состояние?

Да, есть два основных состояния:

Жив (health > 0) - персонаж может:
Получать урон
Получать опыт
Повышать уровень
Взаимодействовать с миром
Мертв (health = 0) - персонаж:
Не может получать урон (логически, хотя технически может)
Вероятно, не должен получать опыт (в текущей реализации может)
Требует воскрешения/респауна
Правила состояний (которые стоит добавить):

Нельзя лечиться, если мертв
Нельзя получать опыт, если мертв
Нельзя повышать уровень, если мертв

# Реализованный класс: Player
```
class Player:
    available_classes = ("warrior", "mage", "archer", "healer")
    max_level = 100
    base_experience = 1000


    def __init__(self, nickname: str, player_class: str, level: int = 1, health: int = 100, experience: int = 0):
        self.nickname = nickname
        self.player_class = player_class
        self.level = level
        self.health = health
        self.experience = experience
```
## Атрибуты класса: 
- `avalible_classes` — Допустимые классы
- `max_level` — Макисмальный уровень
- `base_experience` — Начальный уровень
